0 Understand the data flows first

Two separate flows. Two separate Google Sheets. One GitHub repo for payment records.

Action Goes to Section
Someone pays for a workshop track GitHub CSV (daily file in registrations/) + Google Sheet (Workshop Registrations) Workshop
Someone submits a product enquiry Google Sheet (AI Products Leads) AI Tools
Workshop payment fails / is cancelled Nothing saved — user stays on payment step Workshop
Razorpay is used only for workshop courses.
AI product interest forms have no payment — they only capture name, phone, email, and message.

A Workshop — Registration & Payment

Razorpay → Google Sheet + GitHub CSV file (daily). Uses workshop-script.js.

1

Create the Workshop Google Sheet

Go to sheets.google.com → create a new spreadsheet named "AI Workshop Registrations".

Leave it empty — the script creates the headers automatically on first use.

2

Add the Apps Script

Inside the sheet: Extensions → Apps Script. Delete any existing code.

Open the file workshop-script.js from this repo and paste its entire contents.

Click Save (Ctrl+S).

3

Deploy the Workshop Script as a Web App

Click Deploy → New deployment.

  • Type: Web App
  • Execute as: Me
  • Who has access: Anyone

Click Deploy → Authorise when prompted.

Copy the Web App URL — you'll need it in Step 5.
4

Get a GitHub Fine-Grained Token

This token lets the website write the daily CSV files to your aiproducts repository.

  1. Go to GitHub → Settings → Developer settings → Fine-grained tokens
  2. Resource owner: karnkeshav
  3. Repository access: Only selected repositories → aiproducts
  4. Permissions → Contents: Read and write
  5. Generate token and copy it.

B AI Products — Lead Capture

Form submission → Google Sheet (Leads). Uses leads-script.js. No payment involved.

1

Create the Leads Google Sheet

Go to sheets.google.com → create a second, separate spreadsheet named "AI Products Leads".

Leave it empty — the script creates the headers automatically.

Columns that will be created: Timestamp · Product · Name · Phone · Email · Message

2

Add the Leads Apps Script

Inside the Leads sheet: Extensions → Apps Script. Delete any existing code.

Open the file leads-script.js from this repo and paste its entire contents.

Click Save.

3

Deploy the Leads Script as a Web App

Click Deploy → New deployment.

  • Type: Web App
  • Execute as: Me
  • Who has access: Anyone

Click Deploy → Authorise.

Copy this Web App URL — different from the workshop URL.

C Add all secrets to GitHub (one-time)

Go to github.com/karnkeshav/aiproducts → Settings → Secrets and variables → Actions. Add each secret below:

GOOGLE_SHEET_URL Workshop Apps Script Web App URL (from Part A, Step 3)
RAZORPAY_KEY_ID Your Razorpay publishable key — starts with rzp_live_ or rzp_test_
CSV_GITHUB_TOKEN Fine-grained GitHub PAT from Part A, Step 4
LEADS_SHEET_URL Product Leads Apps Script Web App URL (from Part B, Step 3)
✓

Trigger a deploy

Push any change to main (or go to Actions → Deploy to GitHub Pages → Run workflow).

GitHub Actions will inject all four secrets into config.js and deploy the page.

✓ You're done.
Workshop payments → GitHub registrations/YYYY-MM-DD.csv (appends daily, new file each day) + Google Sheet.
Product enquiries → Google Sheet "AI Products Leads" (one row per enquiry).

D How to verify it's all working

Quick checks after your first deploy.

Workshop Script health-check: paste your GOOGLE_SHEET_URL into a browser. You should see:
AI Workshop registration endpoint is active. Registrations are saved here.

Leads Script health-check: paste your LEADS_SHEET_URL into a browser. You should see:
AI Products leads endpoint is active. Enquiries are saved here.

Test a product enquiry: open your site, tap "I'm Interested" on any product, fill in the form, submit. Check the "AI Products Leads" Google Sheet — a new row should appear within seconds.

Test a workshop payment: use Razorpay's test mode key (rzp_test_…) and test card number 4111 1111 1111 1111. After payment, check:

  • Google Sheet "AI Workshop Registrations" — new row
  • GitHub repo → registrations/YYYY-MM-DD.csv — new file
